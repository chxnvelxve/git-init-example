# Git and REST API Key Takeaways

## Why this matters for my career path
These two topics support my move from mid-level delivery engineer into a stronger automation- and architecture-capable engineer. Git helps me manage files, scripts, and version history. REST APIs help me interact with platforms like Aruba Central in a repeatable way.

## Git takeaways
- Git tracks changes to files over time.
- A repository is the working container for code, notes, and automation artifacts.
- Commits are save points with meaningful history.
- Branches let me test changes safely without breaking the main version.
- Pull helps sync remote changes to my local repo.
- Push sends my local changes to a remote repo.
- Git is useful for storing scripts, templates, notes, and project artifacts.

## Git in VS Code
- VS Code gives me a simple visual way to review changed files.
- I can stage, commit, view diffs, and sync without doing everything in terminal.
- This is likely the easiest way for me to build a real Git habit.
- Later this can support GitHub and Copilot workflows.

## REST API takeaways
- An API is a way for software systems to exchange data.
- A request is what I send to the system.
- A response is what the system sends back.
- GET is used to retrieve data.
- POST is used to submit or create data.
- APIs commonly use headers for things like authentication and content type.
- Tokens are used to prove I am authorized.
- JSON is a structured data format commonly returned by APIs.

## JSON basics
- JSON uses key/value pairs.
- Objects use curly braces `{}`.
- Lists use square brackets `[]`.
- JSON is easy to read and useful for automation and reporting.

## How this connects to Aruba Central

- After that, I can move into Python scripts that pull inventory, AP health, or WLAN data.
- This is the bridge between course learning and my first real automation artifact.
